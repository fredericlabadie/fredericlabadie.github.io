Data engineering

BigQuery pipeline rebuild — ~50% cost reduction across pipeline and dashboards

Three raw scans → one · fifteen CTEs → zero · 1.2 TB intermediate retired · +

Redesigned a GA4 → BigQuery analytics pipeline that had grown into a $112-per-run CTE chain processing 18 TB per execution. The redesign centred on a single shared computation; the result was ~50% cost reduction across both pipeline and dashboard queries, with the 1.2 TB GA4_Event_Params intermediate table retired entirely. Heavy dashboards halved from ~$200 to ~$100 per day. Delivered as a four-phase migration plan with a two-to-four-week validation window.

Cross-platform debugging

QSR error taxonomy — noisy alerts made trustworthy at scale

Hierarchical error-category scheme · cart_id enrichment · reduces false positives in the AI triage layer · +

A major QSR brand's Amplitude alert channel had genuine checkout failures buried in low-traffic noise. Built three layers of remediation: threshold redesign with troubleshooting guidance on each chart, a hierarchical error-category ID scheme (SYS-INF-001, SYS-APP-VAL-010) with governance against schema drift, and cart_id enrichment so error events carry the same context as their success counterparts. Alerts became actionable without detective work, and the taxonomy directly reduces false positives in the AI alert-triage system above.
