Case study · Lens 07 · Data Engineering · Pipeline build

The pipeline layer behind the mobile instrumentation.

Scheduled Dataform pipelines transforming raw GA4 events into analysis-ready BigQuery tables — with query rights transferred to the client at rolloff.

/ State-regulated retail platform · BigQuery + Dataform · production

→
What happened
A state-regulated retail platform now runs production Looker Studio dashboards (including executive revenue-linked product reporting) on a scheduled Dataform pipeline that transforms raw GA4 events into analysis-ready BigQuery tables. Production SQL scripts authored, maintained in the agency-client analytics repo, with query rights formally transferred to the client analytics lead at engagement rolloff — reporting continuity preserved after the engagement ended.
Context
State-regulated retail platform's mobile instrumentation needed an analysis-ready data layer underneath it — raw GA4 events out of BigQuery were not directly usable for the executive Looker Studio dashboards. No production pipeline existed for the new app data, and reporting was at risk of stalling at rolloff if the agency held query ownership.
What I did
Built the scheduled Dataform pipeline producing GA4_Event_Params, GA4_Event_Users, and GA4_Events — the foundation the production Looker Studio dashboards run on. Authored production SQL scripts (bq_product_revenue_inserts.sql, bq_sales_product_data_table_inserts.sql) maintained in the agency-client analytics repo, powering revenue-linked product reporting used in executive readouts. Scoped a separate product-inventory pipeline modelled against existing product tables. Formal query-rights transfer to the client analytics lead at rolloff.
