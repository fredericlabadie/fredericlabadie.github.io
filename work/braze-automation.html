Case study · Topic 06 · Marketing Analytics · Lifecycle automation

Email telemetry that stops being a copy-paste job.

A Python automation against the Braze REST API that survives campaign edits, exclusion rules and all.

/ State-regulated retail platform · Braze REST API · Python

The four-person demand-marketing team was spending ~2 hours per month manually extracting Braze campaign metrics for monthly readouts. Designed and shipped a Python automation that authenticates against the Braze REST API, resolves campaign IDs dynamically by name pattern (so campaign clones and rebuilds don't silently break the report), excludes a known-noisy luxury form-submission campaign from aggregates, and writes send volume, open rate, click rate, unsubscribe rate, and revenue attribution directly to a Google Sheet tab ready for the team's template. Dynamic name-matching with exclusion rules was the key engineering decision — hard-coded IDs would have failed silently on every campaign edit.
