Case study · Lens 06 · Marketing Analytics · Lifecycle automation

Email telemetry that stops being a copy-paste job.

A Python automation against the Braze REST API that survives campaign edits, exclusion rules and all.

/ State-regulated retail platform · Braze REST API · Python

→
What happened
A four-person demand-marketing team went from ~2 hours of manual Braze data extraction per month to zero clicks per cycle — a Python automation against the Braze REST API now writes send volume, open rate, click rate, unsubscribe rate, and revenue attribution directly to the team's Google Sheet template. The key engineering decision: dynamic name-matching with exclusion rules, not hard-coded campaign IDs — so campaign clones and rebuilds don't silently break the report. First automated delivery: May 26.
Context
A four-person demand-marketing team spending ~2 hours per month manually extracting Braze campaign metrics for the monthly readout. The fragility wasn't just time — any automation built on hard-coded campaign IDs would silently fail every time a campaign got cloned or rebuilt, and a known-noisy luxury form-submission campaign needed to be excluded from aggregates without manual triage every cycle.
What I did
Shipped a Python automation against the Braze REST API. Dynamic campaign-ID resolution by name pattern with explicit exclusion rules for the luxury form-submission noise. Writes send volume, open rate, click rate, unsubscribe rate, and revenue attribution directly to a Google Sheet tab ready for the team's template. The engineering decision around dynamic name-matching is what makes it survive campaign edits — the difference between a script that runs and a script that lasts.
