Clean taxonomy, trustworthy alerts, enriched context.

Three layers of remediation on an alert channel where genuine checkout failures were buried in noise from low-traffic Active% triggers.

Core argument

Error events must carry the same context as their success counterparts.

Otherwise alerting can be technically correct but still require detective work before anyone can act on it.

What happened

A noisy alert channel became a more trustworthy operating surface.

A major QSR's Amplitude alert channel was firing on too many low-signal events. Genuine checkout failures were buried in low-traffic Active% noise. I delivered three layers of remediation: redesigned thresholds with troubleshooting guidance (shipped); a hierarchical error-category ID scheme designed and proposed for implementation; and a cart_id enrichment specification so checkout error events would carry the same contextual anchor as their success counterparts.

The taxonomy work directly reduces false positives in the AI alert-triage system because the model has cleaner categories and richer context to reason from.

Context

The alert channel was correct but exhausting.

Low-traffic monitors produced noisy Active% triggers that drowned out genuinely important checkout failures. On-call responders had to do detective work before they could know whether an alert represented a real checkout problem, which basket was affected, or how the error should be categorised.

Task

Make the alert itself carry the start of the diagnosis.

The work needed to reduce false positives while improving diagnostic context. That required changes at the monitor threshold layer, the error taxonomy layer, and the event-schema layer.

// 02 / remediationthresholds / taxonomy / context

Remediation

Three fixes, one operating principle.

Trustworthy alerting depends on threshold design, stable categorisation, and event context that lets engineers move from alert to diagnosis.

1Threshold redesign

Redesigned monitors around absolute rate plus rate-of-change, reducing overreaction to low-traffic Active% volatility.

2Chart-level guidance

Added engineering-facing troubleshooting guidance directly on each chart so the alert carried the start of the diagnosis.

3Error taxonomy

Designed hierarchical error-category IDs with deterministic mapping rules and governance guidelines against schema drift.

4Context enrichment

Authored the specification to add cart_id to Checkout Started Error Received — ready for dev — so error and success events would share the same context level.

Action

The alert channel was redesigned across layers, not just muted.

I redesigned alert thresholds, added troubleshooting guidance to the relevant Amplitude charts, designed a hierarchical error-category ID scheme with governance rules, and authored the specification to enrich checkout error events with cart_id. The goal was not fewer alerts for its own sake. It was alerts that responders could interpret and act on without recreating the context from scratch.

// 03 / resulttrustworthy alerts / cleaner AI triage

Result

Better alerting made the AI workflow safer downstream.

The AI triage system is only as useful as the categories and context it reads. Cleaner upstream taxonomy reduces downstream guesswork.

Outcome

Alerts became more actionable before AI entered the loop.

  • Thresholds redesigned around absolute rate plus rate-of-change.
  • Engineering-facing troubleshooting guidance added directly to charts.
  • Hierarchical error-category ID scheme designed and proposed, with deterministic mapping rules and governance guidelines.
  • cart_id enrichment specified for checkout error events — authored as a ready-for-dev ticket so failures could be correlated to basket instances.
  • False-positive reduction support created for the AI alert-triage workflow.

Design lesson

Alerting problems are often taxonomy problems wearing an operations costume.

If errors are inconsistently categorised or stripped of the context success events carry, the alert channel forces humans to reconstruct basic facts. Fixing that starts upstream in instrumentation and schema governance.
