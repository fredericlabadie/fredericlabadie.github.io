Case study · Topic 02 · Instrumentation · Error taxonomy

Clean taxonomy, trustworthy alerts, enriched context.

Three layers of remediation on an alert channel where genuine checkout failures were buried in noise from low-traffic Active% triggers.

/ Major quick-service restaurant · Amplitude monitors

Three layers of remediation on an alert channel where genuine checkout failures were buried in noise from low-traffic Active% triggers. Error events must carry the same context as their success counterparts — otherwise alerting is correct but actionable only with detective work.
01
Threshold redesign
Redesigned alert thresholds (absolute rate + rate-of-change) and added engineering-facing troubleshooting guidance directly on each chart, so the alert itself carries the start of the diagnosis.
02
Error-category ID scheme
Designed a hierarchical, stable error-category ID scheme (SYS-INF-001, SYS-APP-VAL-010) with deterministic mapping rules and governance guidelines to prevent schema drift over time.
03
Context enrichment
Added cart_id to Checkout Started Error Received events so engineering could correlate specific basket instances to checkout failures — same context level as success events.

The taxonomy work directly reduces false positives in the AI alert triage system — see the AI workflow case →.
