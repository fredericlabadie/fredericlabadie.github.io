Case study · Topic 02 · Instrumentation · SDK debugging

A silent iOS data-loss bug, traced to the initialisation sequence.

An iOS undercount of Application Opened events vs. Android — root-caused to a Segment SDK 1.9.1 gate that silently drops events when enabled = false.

/ Major quick-service restaurant · Segment · analytics-swift 1.9.1

iOS was systematically undercounting Application Opened events compared to Android. Investigation traced the root cause to the app's initialisation sequence: rawSegment.enabled = false at startup, re-enabled only after remote config completed. The Segment SDK 1.9.1 process(incomingEvent:) returns immediately when enabled == false — cold-launch events fired before the gate flipped were silently dropped, not queued.

Same engagement also surfaced cross-platform tracking-plan parity gaps (web on legacy dataLayer/GA naming; apps on a unified Title Case Segment model) and a GTM misconfiguration where two international locales were feeding the main analytics container.
