Case study · Experimentation · Conversion optimisation

The quiz worked. The question was where to put it.

A hardware authentication brand had a product selection quiz quietly converting users — but it was under-surfaced. Phase 1 used GA instrumentation to confirm the signal. Phase 2 used VWO to A/B test four store-page placement variants. Prominence was the variable, not subtlety.

/ B2B/B2C hardware authentication brand · conversion optimisation · 2016

Role-fit signal
The quiz signal was already in the data. The job was to find it, then test the fix. The Philips role describes designing measurement that drives product decisions. This case shows that method applied to UX placement: Phase 1 instrumented the conversion signal; Phase 2 turned the hypothesis into a controlled test. The approach — instrument first, experiment second — transfers directly to feature telemetry and checkout-flow optimisation.
→
What happened
A hardware authentication brand sold a range of security keys across form factors and connectivity standards. A product selection quiz existed to help users navigate the complexity — but it lacked a prominent entry point. Phase 1 GA analysis confirmed quiz-exposed users consistently outconverted non-exposed users. Phase 2 VWO A/B tested four store-page placement variants. The thick-bar entry-point variant generated roughly twice the pathed CVR of the control and outperformed all alternatives across new and returning cohorts.
The quiz was already converting. The data confirmed it. The test found where it needed to be visible.
Context
Wide product range, high-consideration purchase, no clear entry path from the main store page. A product selection quiz had been built to resolve the navigation friction but was embedded without a prominent entry point. Purchase-intent traffic was landing on the store page and largely bypassing it. The initial brief was general conversion optimisation; Phase 1 data reframed the question.
Task
Analytics instrumentation and insights: tracking the quiz flow and conversion pages in GA, analysing the behavioural split between quiz-exposed and non-exposed sessions, implementing VWO for Phase 2, and interpreting the results. The UX team designed the visual variants; the measurement layer and test framing were the analytics contribution.
Action
Phase 1: GA instrumentation across Start, Quiz, Setup, and product category pages. Cohort analysis comparing quiz-exposed vs non-exposed session conversion rates. Phase 2: VWO A/B test across four store-page entry variants — control (no bar), thick-bar, thin-bar subtle, thin-bar bright. Custom GA dimensions captured test assignment against downstream conversion events for pathed CVR analysis, not just click-through.
Result
Phase 1 confirmed the quiz signal: exposed sessions outconverted non-exposed sessions, which declined over the same period. Phase 2 confirmed placement as the lever: the thick-bar variant generated roughly twice the pathed CVR of control. Both thinner variants underperformed, confirming that visual weight was the variable. Both new and returning cohorts responded.
01
Phase 1 instrumentation
GA instrumentation across quiz entry, quiz flow, and conversion pages. Established the measurement baseline to compare quiz-exposed and non-exposed session behaviour.
02
Phase 1 analysis
Cohort analysis confirmed quiz-exposed users outconverted non-exposed users. Non-exposed session conversion declined over the same period. The quiz was the signal; placement was the gap.
03
Phase 2 VWO test setup
VWO configured for four store-page placement variants. Custom GA dimensions mapped test assignment to downstream conversion events for pathed CVR analysis by variant and by new/returning cohort.
04
Phase 2 results and recommendation
Thick-bar variant — sized to a hero image — won across both cohorts. Subtler variants underperformed, confirming visual prominence as the variable. Placement recommendation made on data, not aesthetic preference.
