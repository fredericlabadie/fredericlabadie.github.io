Case study · Path 02 · Analytics platform migration

Off Adobe, onto Google. Three phases and a cleared backlog.

A D2C brand was running Adobe Analytics on a Meteor.js SPA. Every tracking change required developer lift, so requirements had accumulated for years. The migration to GA/GTM required solving a non-standard data layer problem first — then it unlocked a backlog clearance and a round of capabilities Adobe had made impractical.

/ D2C health & beverage brand · Adobe Analytics to GA migration · 2019

Role-fit signal
Adobe made every analytics change a developer ticket. Requirements accumulated. The Philips role describes translating data needs into event specifications, implementation requirements, and production telemetry. This case shows what that looks like when the existing platform has made requirements undeliverable: the migration wasn’t just a tool change — it was the precondition for a backlog clearance and a more maintainable measurement architecture.
→
What happened
A D2C health and beverage brand was running Adobe Analytics on a Meteor.js Single Page Application. The platform required heavy developer involvement for every analytics change, so tracking requirements had accumulated without shipping. The brief was to migrate to GA via GTM. The challenge was a non-standard Meteor.js pre-render pattern that required a custom data layer push architecture. Three phases followed: migrate the Adobe implementation, clear the deferred requirements backlog, then add capabilities the platform had never supported.
A measurement platform that requires developer involvement for every change is not a measurement platform — it is a queue.
Context
The Meteor.js SPA pre-rendered a shell before rendering page content, which meant DOM-ready triggers fired before relevant elements existed. Standard GTM trigger logic was unreliable. Adobe Analytics had compounded this by requiring developer involvement for every tracking change — tags couldn’t be managed independently. A backlog of deferred requirements had accumulated over years: AdWords tracking, partially-implemented technologies, and enhancements that had been deprioritised because the overhead made them impractical.
Task
Full implementation: audit the Adobe implementation, design the Meteor.js data layer push architecture, write and build GTM tags, migrate applicable Adobe components, clear the requirements backlog across phases, and ship Phase 3 capabilities the client had not originally specified. QA throughout.
Action
Designed a data layer push architecture — custom events and variables pushed from the application to the GTM data layer, so tags fired only after page content was confirmed loaded. Phase 1 migrated Adobe to GA. Phase 2 cleared deferred requirements including AdWords and partially-functional tech. Phase 3 added Facebook Ads conversion tracking, JSON-LD SEO schema, and additional enhancements now possible because the platform supported rapid iteration.
Result
Adobe Analytics replaced by GA with full feature parity. Meteor.js pre-render problem solved. Backlog cleared. Facebook conversion tracking, JSON-LD schema, and additional capabilities shipped. Operating model changed: analytics requirements no longer bottlenecked by development cadence for the majority of tracking updates.
01
Audit and data layer design
Reviewed the Adobe implementation, identified reusable components. Designed the Meteor.js data layer push architecture: custom events and variables from the application, fired only after content confirmed loaded.
02
Phase 1 — Platform migration
Migrated Adobe Analytics to GA. Reused applicable webhooks and code components. Established GA as the tracking baseline with full data layer architecture in place.
03
Phase 2 — Backlog clearance
Cleared deferred requirements: AdWords tracking, partially-implemented technologies that had been rolled out but were not functioning correctly. Requirements that previously needed developer tickets now managed independently in GTM.
04
Phase 3 — Proactive enhancements
Added capabilities the client had not originally specified: Facebook Ads conversion tracking, JSON-LD SEO schema, and additional layers now possible because the platform supported rapid iteration without developer involvement.
