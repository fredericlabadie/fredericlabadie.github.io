Case study · Path 02 · SDK debugging & cross-platform parity

A silent iOS data-loss bug, traced to the initialisation sequence.

Human-led anomaly detection flagged a systematic iOS undercount of Application Opened events versus Android; AI-assisted SDK source review helped trace the runtime behaviour causing silent cold-launch event loss.

/ Major quick-service restaurant · Segment · analytics-swift 1.9.1

Role-fit signal
Runtime parity beats documentation parity. UX telemetry only works when product teams can trust the counts underneath the dashboard. This case shows the implementation discipline behind that trust: compare platforms, gather proof, read the startup sequence, inspect the SDK path, and separate user behaviour from measurement failure.
→
What happened
A major QSR had a systematic iOS undercount of Application Opened events versus Android. The bug was silent: no visible errors, no obvious dropped-event logs, only a stable undercount. I traced the issue to the app's initialisation sequence and Segment analytics-swift 1.9.1: process(incomingEvent:) returns immediately when enabled == false, so cold-launch events fired before remote config re-enabled the SDK were dropped rather than queued.
Telemetry that undercounts silently is worse than telemetry that is missing. A missing event can be investigated. A silent undercount looks trustworthy while skewing every downstream funnel.
Context
Mobile analytics showed a stable difference between iOS and Android Application Opened counts. Downstream funnel reading was being skewed, but there was no explicit error pointing to the source of the loss. The undercount looked like real user behaviour.
Risk
Because the loss was silent, stakeholders could read the difference as platform behaviour rather than measurement failure. That made root-cause discipline essential: read the app startup sequence and the SDK source path actually running in production.
Method
First, I identified the cross-platform count gap and gathered proof that the issue was not a dashboard mismatch. Then I used AI assistance to inspect the Segment analytics-swift source path, compare candidate failure modes, and narrow where the cold-launch event loss lived.
Result
The issue moved from suspected platform difference to explainable data loss. The same engagement also surfaced cross-platform tracking-plan parity gaps and a GTM container governance issue where two international locales were feeding the main analytics container.
